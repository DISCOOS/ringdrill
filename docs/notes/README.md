# Notes

Episodic engineering knowledge: things we learned by trying, that are worth remembering but do not rise to an ADR (a decision) or a section in `architecture.md` (the current state).

Each note captures a specific investigation, gotcha, or experimental result. Format is loose but usually includes:

- Date the finding surfaced
- Context (why we were looking)
- What we tried
- What we found
- Implications (what to do or not do because of this)

Notes are additive. Do not rewrite older notes when new information arrives — add a new note instead, or add a "See also" pointer at the bottom of the older one.
