3. **Brief default audience should be Øvelsesleder, not Deltaker (DESIGN-004).** The brief always opens on `participant`. In practice only staff (øvelsesleder, veileder) use the app — there are no observed cases of participants using it themselves — so default to `director` (full content). The PII exposure is to trusted staff on the local device, so the default is acceptable; the finer control between staff roles is item 4.

4. **App-user role selector, staff-only (DESIGN-004 + DESIGN-006 Roster axis).** Since participants do not use the app, `participant` stays an export/print brief audience, not an app-user role. The selector only needs the two staff roles, Øvelsesleder and Veileder, and its real job is gating actor PII: `director` shows `realName`/`phone`, `instructor` does not. Let the device user declare which staff role they hold to set the default brief audience (subsumes item 3). This is about who holds *this* device, distinct from the Roster/Bemanning staffing of *other* people and from the ADR-0019 session role (coordinator/observer/roleplayer). Park as a small local preference for now; it converges with the identity model (ADR-0024/0025) and the staffing axis later.
