`buildRouter` currently mounts five shell tabs and opens detail routes through `_ContextSheetDeepLinkLauncher` / `_BriefDeepLinkLauncher`; `/i/` and `/o/` redirect into Program before opening their flows. `MainScreen._initTab` and `_onDestinationSelected` currently use un-prefixed route constants directly.
`ProgramService.loadProgram(uuid)` can validate a canonical program id and `setActive(uuid)` activates it; entity lookups remain scoped to the active program. The stage-1 `_stationListController`, `_rolePlaysController`, and `_teamsPageController` fields must remain injected into the Program page after their standalone tabs are removed.

Stage 2 landed with canonical `/program/:uuid/...` paths, one URL activation gate, active-relative redirects for old paths, and a two-tab Program + Map shell. Canonical detail leaf routes are intentionally flattened under the program route so each URL mounts exactly one sheet launcher; the URL shapes still follow ADR-0032. The structural controllers remain field-held for Program segments, and stage 4 can add Roster beside the surviving two destinations.
